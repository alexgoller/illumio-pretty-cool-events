How It Works

Overview

Pretty Cool Events continuously monitors your Illumio PCE for audit events and routes them to notification channels. It polls the PCE Events API at a configurable interval, matches each event against your watcher rules, and dispatches matched events to one or more plugins.

Illumio PCE → Event Polling → Watcher Matching → Plugin Dispatch → Notifications

Getting Started

1
Configure PCE Connection

Go to Configuration and enter your PCE hostname, API credentials, and organization ID. The poll interval controls how frequently events are fetched (default: 10 seconds).

2
Enable and Configure Plugins

Go to Plugins and enable the notification channels you want (Slack, Email, PagerDuty, etc.). Each plugin has its own credentials and settings.

3
Create Watchers

Go to Watchers and define which events trigger which plugins. Use exact event types, wildcards (user.*), or catch-all (.*).

4
Monitor

Use the Dashboard and Statistics to track event flow. Use the Event Explorer to browse historical events and fine-tune your watchers.

Watcher Matching

Each watcher has an event pattern and one or more actions. When a PCE event arrives:

  1. Pattern match — The event type is checked against exact patterns first (O(1) lookup), then regex patterns.
  2. Status filter — The event status is checked. Use * to match any status (including null).
  3. Field matching — If the watcher has match_fields, each named field is checked against the event.
  4. Dispatch — Matched events are rendered through a template and sent to the configured plugin.
Pattern Examples
PatternMatches
user.loginExactly user.login events
user.*All user events (login, logout, create, etc.)
rule_set.*Rule set create, update, delete
agent.*All agent events (activate, tampering, etc.)
request.*Authentication failures, server errors
.*Every event (catch-all)
Status Matching
ValueBehavior
successOnly successful events
failureOnly failed events
* or anyAll events regardless of status (including null)
Advanced: Field Matching

Add match_fields to a watcher's extra_data to filter on any event field using dot notation:

watchers:
  ".*":
    - status: "*"
      plugin: PCESlack
      extra_data:
        template: default-slack.html
        channel: "#security-alerts"
        match_fields:
          severity: "err|warning"              # err OR warning
          created_by.user.username: "admin@.*"  # regex on nested field
          severity: "!info"                     # NOT info

Supported operators: exact match, * (any), ! (negation), | (alternatives), regex patterns, and dot-notation for nested fields.

Templates

Events are formatted using Jinja2 templates before being sent to plugins. Each watcher can specify a different template.

TemplateFormatBest for
default.htmlPlain textConsole, general use
default-slack.htmlSlack Block Kit JSONSlack messages
default-json.htmlJSONFile logging, webhooks
email.tmplHTMLEmail notifications
sms.tmplShort textSMS / PagerDuty
default-teams.tmplMarkdownMicrosoft Teams

Templates have access to all event fields as top-level variables (e.g. {{ "{{ event_type }}" }}) and also as a nested event object (e.g. {{ "{{ event.created_by.user.username }}" }}).

CLI Reference

pce-events run --config config.yaml          # Start the service
pce-events config init                        # Create a new config interactively
pce-events config validate --config config.yaml  # Validate config
pce-events config show --config config.yaml   # Show config (secrets masked)
pce-events watcher list --config config.yaml  # List all watchers
pce-events watcher add --config config.yaml   # Add a watcher interactively
pce-events test-plugin PCEStdout --config config.yaml  # Test a plugin
